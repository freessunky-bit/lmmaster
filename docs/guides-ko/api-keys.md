# 로컬 API 키 발급 가이드 (한국어)

## 키가 필요한 이유

LMmaster의 로컬 게이트웨이는 동일 PC의 모든 프로그램이 무인증으로 호출하지 못하도록 항상 키를 요구합니다(ADR-0007, 보안 위험 S1 대응).

## 발급 절차

1. 로컬 API 화면 → [+ 새 키 발급].
2. 별칭(필수), 프로젝트 바인딩(선택), scope(전부 / 특정 모델 glob), 만료일(선택), rate limit(선택)을 설정.
3. 발급 즉시 평문이 1회 표시됩니다 — 이후에는 표시되지 않습니다. 클립보드 복사 후 사용처에 보관하세요.

## scope 모범 예시

- 외부 웹앱이 채팅만 사용: `endpoints: ["/v1/chat/*"]`, `models: ["qwen-*", "llama-*"]`.
- 다른 도구가 임베딩만 사용: `endpoints: ["/v1/embeddings"]`.
- admin 권한(설치/관리)은 GUI에서 자동 발급되며, 외부 키에는 부여하지 않습니다.

## 분실 시

[revoke]로 즉시 무효화 후 새 키 발급. 기존 키의 사용 로그는 별도로 보관됩니다.
