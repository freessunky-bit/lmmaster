# 8. 한국어 UI 정보 구조 (IA)

> 산출물 #8. 9개 화면의 한국어 카피, 정보 우선순위, 상태/오류 메시지 톤.
> 디자인 토큰과 컴포넌트는 `docs/design-system-plan.md`(산출물 #9) 참조.

## 8.1 글로벌 셸

```
┌──────────────────────────────────────────────────────────────────┐
│  [≡] LMmaster                              [⌘K]  [상태]  [설정]  │  ← Topbar
├──────────┬───────────────────────────────────────────────────────┤
│          │                                                       │
│  홈      │                                                       │
│  카탈로그│                                                       │
│  설치    │                  Main Content Area                    │
│  런타임  │                                                       │
│  프로젝트│                                                       │
│  키      │                                                       │
│  워크벤치│                                                       │
│  진단    │                                                       │
│  설정    │                                                       │
│          │                                                       │
└──────────┴───────────────────────────────────────────────────────┘
```

- 좌측 사이드바: 9개 메뉴(아이콘 + 한글 라벨, 접힘 가능).
- 상단 topbar: 앱명, command palette(⌘K), gateway 상태 인디케이터, 설정 진입.
- 콘텐츠는 카드/패널 기반.
- Command palette는 모든 메뉴 + 빠른 액션(예: "모델 설치", "키 발급", "로그 열기")을 한국어 라벨로 검색.

## 8.2 화면별 IA

### 8.2.1 홈 / 추천

목적: 실행 직후 사용자가 "내 PC 상태 + 무엇을 설치할지"를 5초 안에 파악.

상단부터 순서대로:
1. **환영 카드** — "안녕하세요, 어떤 모델로 시작해볼까요?" (간결, 마케팅 톤 금지).
2. **내 PC 사양 카드** — OS, CPU, RAM, GPU, VRAM, 디스크 여유. mono + tabular-nums.
   - 우측에 "다시 점검" 버튼.
3. **추천 모델 3종 카드** (가로 3분할):
   - **상위 품질** (best_choice): 최고 품질, 자원 충분 시.
   - **균형형** (balanced_choice): 품질·속도·자원 균형.
   - **경량형** (lightweight_choice): 자원 절약 우선.
   - 각 카드: 모델명 / 카테고리 / 권장 사유 1줄 / 예상 VRAM·RAM·디스크 / [설치] 버튼.
4. **Fallback 카드** — "지금 자원이 부족하다면" 옵션.
5. **빠른 액션** — "기존 웹앱 연결", "키 발급", "로그 보기".

상태:
- 미설치 상태: 큰 [추천 모델 설치하기] CTA.
- 설치 진행 중: 진행률 카드(설치 센터로 링크).
- 사용 가능: "사용 가능" 뱃지.

### 8.2.2 모델 카탈로그

목적: 카테고리별 탐색·비교·설치.

레이아웃:
- 좌측 필터 패널: 카테고리(에이전트/캐릭터/코딩/사운드-STT/사운드-TTS/SLM/임베딩-rerank), 사양 적합도(내 PC 기준), 라이선스(상업/비상업), 태그(핫/추천/실험용).
- 메인: 카드 그리드. 각 카드:
  - 모델명 + 카테고리 뱃지 + 핫/추천/실험용 태그.
  - 짧은 설명 1~2줄.
  - 권장 양자화, 최소·권장 VRAM/RAM, 설치 크기.
  - 적합도 라이트(녹/황/적) — "내 PC에 적합/주의 필요/부적합".
  - [상세] [비교에 추가] [설치] 버튼.
- 상단 검색바 + 정렬(최신/추천/이름).

비교 기능: 최대 4개 모델을 가로로 비교. 동일 사양 셀은 비교 강조.

### 8.2.3 설치 센터

목적: 다운로드 큐 + 진행률 + 오류 복구.

- 상단 탭: "설치 중 / 대기 중 / 완료 / 오류".
- 각 행:
  - 모델명, 어댑터, 양자화.
  - 진행률 바 + mono 숫자(%, 다운로드 속도, ETA).
  - 단계 표시: 다운로드 중 → 검증 중 → 압축 해제 → 워밍업 중 → 사용 가능.
  - [일시정지] [취소] [재시도] [폴더 열기].
- 오류 발생 시: 한국어 사용자 메시지 + 원문 메시지 expandable + 추천 복구 액션 버튼(예: "체크섬 불일치 — 다시 다운로드").

### 8.2.4 런타임 / 엔진

목적: 어댑터 관리.

- 어댑터 목록(llama.cpp / KoboldCpp / Ollama / LM Studio / vLLM):
  - 설치 상태(설치됨/미설치/외부 attach).
  - 버전, 빌드 타깃(예: "CUDA 12.4 / Vulkan").
  - 포트(우리가 점유한 내부 포트, 외부 노출 X).
  - 헬스 상태(녹/황/적).
  - [설치] [업데이트] [재시작] [중지] [로그].
- 우측 패널: 선택한 어댑터의 capability matrix(비전/툴/구조화 출력/임베딩).

### 8.2.5 프로젝트 연결

목적: 기존 웹앱 + 다른 웹앱 바인딩, endpoint/SDK 예제 확보.

- 좌측: 프로젝트 목록(이름, 호스트, 키 1개 이상).
- 메인:
  - 프로젝트 메타(이름, 설명, 호스트 — `https://...`).
  - 발급한 키 목록(scope, 마지막 사용 시각, [복사][revoke]).
  - **연결 정보 패널**:
    - Base URL: `http://127.0.0.1:<port>/v1` (복사 버튼).
    - 권장 SDK 호출 예제(JS/TS, Python, curl) — 한국어 주석 포함.
    - "기존 웹앱 연동 가이드 보기" 링크 → `guides-ko/webapp-integration.md`.

### 8.2.6 로컬 API / 키 관리

목적: 키 생애주기.

- 키 목록 표:
  - 별칭, 생성일, 마지막 사용, scope(모델 glob, endpoint glob), 사용량(요청 수/토큰 수), [revoke].
- [+ 새 키 발급] 버튼 → 모달:
  - 별칭(필수), 프로젝트 바인딩(옵션), 모델 scope(전부/특정), 만료일(옵션), rate limit(옵션).
  - 생성 즉시 1회 표시 + 클립보드 복사(이후 표시되지 않음을 안내).
- 사용 로그 뷰(키별):
  - 시각, 모델, 토큰 in/out, 응답 시간. mono.

### 8.2.7 워크벤치 (placeholder)

목적: 미래 확장 자리. v1은 비활성.

- 빈 상태 화면: "곧 제공" + 로드맵 카드(SFT → 양자화 → export → 패키징).
- 베타 신청 옵션(이메일 입력) — v1은 단순 안내.

### 8.2.8 진단 / 로그

목적: 문제 해결.

- 상단 빠른 진단 카드: gateway / 런타임 / 디스크 / 네트워크 / Gemini(opt-in) 상태.
- "환경 점검" 버튼: hardware probe + 런타임 헬스 + 모델 무결성을 1키로 확인.
- 로그 뷰어:
  - 컴포넌트 필터: 설치 / runtime / gateway / hardware / system.
  - 레벨 필터: error / warn / info / debug.
  - 시간 범위.
  - 텍스트 검색 + 정규식 토글.
- Troubleshooting 카드(상위 시나리오):
  - "응답이 느려요"
  - "GPU를 못 잡아요"
  - "다운로드가 자꾸 끊겨요"
  - "기존 웹앱이 연결을 못해요"
  각 카드: 원인 후보 + 자동 진단 + 권장 액션.

### 8.2.9 설정

섹션:
- **일반**: 언어(ko-KR 기본, 영어는 v2 placeholder), 자동 시작, 자동 업데이트.
- **워크스페이스**: 워크스페이스 경로, 모델 저장 경로, 캐시 경로, 캐시 정책(LRU 한도).
- **네트워크**: gateway 포트, 외부 접근 허용(default OFF/금지 명시), proxy.
- **런타임**: 기본 어댑터, GPU 가속 옵션, 동시 실행 제한.
- **AI 도우미 (Gemini)**: 토글, API 키 입력(secrets DB), 사용량 표시.
- **개발자**: 디버그 로그, gateway raw 로그, IPC 로그.
- **실험 기능**: vLLM, Anthropic shim, 워크벤치 베타 등.

## 8.4 오류·상태 메시지 톤

원칙:
- 한국어 사용자 설명을 1차로, 원문 메시지는 expandable.
- 무엇이/왜/다음에 무엇을 — 3요소를 가능하면 한 화면에.
- 마케팅 톤·과장 금지. "걱정 마세요" 류는 1회 이하.
- 사용자가 손해 볼 수 있는 액션(취소/삭제/revoke)은 한 번 더 묻기.

예시:

```
[설치 실패]
모델 파일 검증에 실패했어요.
다운로드된 파일이 손상되었을 수 있어요.

원인 후보
- 네트워크가 도중에 끊겼을 가능성
- 디스크 공간 부족(현재 여유 12.3 GB, 필요 14.0 GB)

추천 액션
[다시 다운로드]   [디스크 정리]   [상세 로그]

원본 메시지 ▼
sha256 mismatch: expected ab12.., got cd34..
```

```
[gateway 연결 끊김]
LMmaster의 로컬 API에 연결하지 못했어요.
방화벽이나 다른 프로그램이 포트를 점유 중일 수 있어요.

[자동 복구]   [포트 변경]   [도움말 보기]
```

```
[GPU 미감지]
GPU 가속을 사용할 수 없어요.
드라이버가 설치되지 않았거나 버전이 낮을 수 있어요.

내 환경
GPU: NVIDIA RTX 4060 Ti
드라이버: 535.x  (권장 550.x 이상)

[드라이버 안내 보기]   [CPU 모드로 진행]
```

## 8.5 i18n 키 규칙

- 키는 `<screen>.<section>.<element>` 점 표기.
  - 예: `home.recommend.cta.install`, `catalog.filter.category.coding`.
- 한국어 카피는 `apps/desktop/src/i18n/ko.json`이 source of truth.
- 영어 자리(en.json)는 v1에서 빈 문자열, v2에 채움.
- 카피 변경 PR은 `voice.md` 가이드 준수 체크.

## 8.6 접근성

- 모든 인터랙션 키보드만으로 가능.
- 포커스 링은 네온 그린 outline + 4px offset.
- 색만으로 상태 구분 금지(아이콘 + 라벨 병기).
- 한국어 mono 폴백(d2coding, jetbrains mono + naver malgun) — 디자인 시스템에서 선언.
