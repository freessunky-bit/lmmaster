# ADR-0002: Desktop shell로 Tauri 2 채택

- Status: Accepted
- Date: 2026-04-26

## Context
Control Plane은 풍부한 정보 밀도의 GUI가 필요하고, 동일 프로세스에서 Rust 기반 supervisor/gateway를 함께 띄워야 한다. 기존 웹앱의 React/TS 자산을 재사용하고 싶다.

## Decision
Desktop shell로 **Tauri 2**를 채택한다. Frontend는 React + TypeScript + Vite, Backend(메인 프로세스/IPC)는 Rust.

## Consequences
- 번들 크기와 메모리 사용량이 Electron 대비 작다. portable 배포 친화적.
- Rust 코어와 동일 프로세스에서 native HTTP gateway, 자식 프로세스 supervisor, 파일 시스템/하드웨어 probe를 수행할 수 있다.
- WebView 차이(Win: WebView2, mac: WKWebView, Linux: WebKitGTK)에서 오는 사소한 CSS/feature 차이는 디자인 시스템에서 수렴 시킨다.
- Tauri 2의 capability/permission 모델(ACL)을 사용해 IPC 표면을 최소화한다.
- 기존 웹앱 코드를 거의 그대로 import해 동일한 디자인 시스템 패키지를 공유한다.

## Alternatives considered
- **Electron**: 성숙도 높지만 번들 크기/RAM이 portable 제품에 불리. 거부.
- **Wails (Go)**: 가능하지만 우리 native 코드 언어 선택(ADR-0003)과 정렬 위해 거부.
- **순수 native (Win32/AppKit/GTK)**: 멀티 OS 비용 폭증. 거부.
